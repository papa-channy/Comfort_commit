귀하의 데이터는 선택한 기간 동안만 보관되며,
지정된 보관 기간 경과 시 자동으로 파기됩니다.
귀하는 언제든지 설정에서 보관 기간을 변경하거나,
보관된 데이터를 직접 삭제할 수 있습니다.

“LLM이 생성한 결과물은 본 서비스의 알고리즘에 의해 생성되며,
그 소유권은 본 서비스에 있습니다.” |
| 유저가 수정한 경우 |
→ 수정 전후 버전 기록
→ 수정 후 결과물에 대해 유저와 공동저작권 구조도 가능 |
| 상업적 활용 |
→ 우리가 PR, 블로그, 전시 등에 활용 가능하도록
→ “생성된 커밋 메시지는 내부 개선 및 마케팅에 활용될 수 있습니다” 명시

✅ Notion의 실제 법적 처리 방식 요약
법적 요구사항	Notion이 처리하는 방식
📜 보관 기간 고지	
→ 이용약관 및 Privacy Policy 내 명시	
→ “고객의 콘텐츠는 고객의 통제 하에 있다”	
→ 즉, 보관 기간은 너희가 정하고 삭제도 너희가 한다는 구조	
→ ⚠︎ 책임을 고객에게 이전	
✅ 삭제 요청 처리	
→ Help 페이지에서	

“We will delete your data upon request within a reasonable timeframe.”
→ 직접 요청하면 처리함. 자동화는 아님. |
| 🔐 목적 외 사용 금지 |
→
“Your content is never used to train AI models.”
→ LLM 훈련, 내부 QA, 제3자 API 공유 없음 (단, 로그는 예외) |
| ⏳ 지체 없는 삭제 |
→ 탈퇴 시 전면 삭제
→ 탈퇴/삭제 요청 없이 그냥 두면 “그대로 영구 저장”
→ 유저가 정한 retention 없으면 무기한 |

분류	예시	설명

✅ 고객이 업로드한 레포에서 가져온 코드	private repo에서 추출된 함수, 파일, diff 등	→ 소유권이 사용자에게 있음
→ 보관/가공 시 동의 필요		
✅ private repo 분석 후 저장된 구조 정보	import graph, 호출 관계, readme 요약 등	원본 repo가 private이면 유출 위험 존재
→ 삭제 요구 시 삭제해야 함
